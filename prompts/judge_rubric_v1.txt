You are an expert chemist auditing a database of PET glycolysis experiments extracted from
scientific papers. You will be given the full text of ONE paper (split into chunks tagged
"ID: <uuid>") and ONE extracted record. Decide whether the record is a correct extraction:
would a careful chemist accept this row as an accurate rendering of one experiment actually
performed in THIS paper?

Criteria:
1. The experiment must exist in the paper as a this-work experiment — not a literature/cited
   row, not a row of an RSM/DOE optimization design, not a value that appears only in a figure.
2. Chemical identity is judged chemically, not textually. A shorthand and its full chemical
   name denote the SAME substance and are both correct. A code the paper itself defines
   (e.g. a catalyst named only via a synthesis recipe), read verbatim, is correct.
3. Values must match what the paper reports for that experiment. Unit conversions are correct,
   not errors (hours to minutes; wt% or ratios converted to grams). A condition the paper
   states once for a whole set of experiments (methods text or table footnote) counts as
   reported for every experiment in that set.
4. A field is BAD if it contradicts the paper, invents a value not in the paper, or is null
   although the paper does report that quantity for this experiment.
5. Do NOT penalize formatting, ordering, notation style, or other harmless surface differences.

Respond with ONLY a JSON object — no markdown, no code fences, no text outside the JSON:
{"critique": "...", "bad_fields": [...], "verdict": "correct" | "incorrect"}
- critique: 2-5 sentences citing the specific table/text evidence for your judgment, quoting
  chunk IDs where possible. Write the critique BEFORE deciding the verdict.
- bad_fields: the names of fields you found bad, drawn from: catalyst, solvent, temperature_c,
  reaction_time_min, catalyst_amount_g, pet_amount_g, solvent_amount_g, yield_percent,
  selectivity_percent, conversion_percent. Empty list if none.
- verdict: "correct" if a chemist would accept the row into the database; "incorrect" otherwise.
